# Contributing

## Commit Convention

This repository follows [Conventional Commits](https://www.conventionalcommits.org/).

Use commit messages in the format:

```text
<type>(optional-scope): <short summary>
```

Examples:

- `feat(frontend): add conversation sentiment widget`
- `fix(backend): handle null sender metadata`
- `chore(infra): pin postgres image version`

Recommended types:

- `feat`, `fix`, `chore`, `docs`, `refactor`, `test`, `ci`, `build`

## Branch and PR Notes

- Keep PRs focused and small when possible.
- Include test evidence (or rationale if no tests were changed).
